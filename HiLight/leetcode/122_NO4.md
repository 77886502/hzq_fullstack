# LeetCode：122.买股票的最佳时机II | 刷题打卡
给定一个数组，它的第 i 个元素是一支给定股票第 i 天的价格。
设计一个算法来计算你所能获取的最大利润。你可以尽可能地完成更多的交易（多次买卖一支股票）。

**注意：你不能同时参与多笔交易（你必须在再次购买前出售掉之前的股票）。**

[来源：力扣（LeetCode）](https://leetcode-cn.com/problems/best-time-to-buy-and-sell-stock-ii)

示例1：
```
输入: [7,1,5,3,6,4]
输出: 7
解释: 在第 2 天（股票价格 = 1）的时候买入，在第 3 天（股票价格 = 5）的时候卖出, 这笔交易所能获得利润 = 5-1 = 4 。
     随后，在第 4 天（股票价格 = 3）的时候买入，在第 5 天（股票价格 = 6）的时候卖出, 这笔交易所能获得利润 = 6-3 = 3 。
```
示例 2:
```
输入: [1,2,3,4,5]
输出: 4
解释: 在第 1 天（股票价格 = 1）的时候买入，在第 5 天 （股票价格 = 5）的时候卖出, 这笔交易所能获得利润 = 5-1 = 4 。
     注意你不能在第 1 天和第 2 天接连购买股票，之后再将它们卖出。
     因为这样属于同时参与了多笔交易，你必须在再次购买前出售掉之前的股票。
```

示例3：
```
输入: [7,6,4,3,1]
输出: 0
解释: 在这种情况下, 没有交易完成, 所以最大利润为 0
```
提示：

- 1 <= prices.length <= 3 * 10 ^ 4
- 0 <= prices[i] <= 10 ^ 4

## 思路分析
股票要想获得最大利润，必须卖出价格大于买入价格 `buy > sell`，且题目规定只能一买一卖。本题我采用了贪心算法的思想，确定股票的最大利润 `maxProfit`。

**怎么确定股票的最大盈利情况呢？** 举个🌰

[1,2,3,4,2,6,1,2,3] 的最大利润过程（只限一次买卖的情况下） ？

你下意识的做法可能是记录让后面的数和前面的数比较记录过程中的最大利润 `maxProfit`，对于这种情况当然适用。

那么多笔交易情况该如何考虑呢？

其实股票的盈利过程是拆分成这样的 `1 => 2，2 => 3，3 => 4， 1 => 6 maxProfit = 5`。我们可以将 `1 => 6` 获得最大利润过程拆分成一个个盈利积累的过程，但只记录其中盈利的部分，即可得到最大利润 `maxProfit`。

## AC代码
```C
int maxProfit(int* prices, int pricesSize){
    int max = 0;
    for(int i=1; i<pricesSize; i++)
    {
        if(prices[i]-prices[i-1]>0)
            max = max + prices[i]-prices[i-1];
    }
    return max;
}
```
时间复杂度：O(n)，空间复杂度：O(1)。

## 总结

学会拆分过程也贪心算法中十分重要的一步。
